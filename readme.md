# ETL Pipeline для проекта прогнозирования оттока клиентов (Telecom Churn)

Данный скрипт реализует классический ETL-процесс (Extract, Transform, Load) для подготовки исторических данных о клиентах телеком-оператора. Собранный датасет предназначен для последующего обучения моделей машинного обучения (ML) в задаче бинарной классификации оттока.

## 🛠️ Архитектура ETL

Процесс разделен на три изолированные функции, что обеспечивает модульность, простоту тестирования и поддержки:

### 1. Извлечение данных (`extract`)
* **Источник:** Реляционная база данных PostgreSQL (Yandex Cloud).
* **Логика:** Соединяет 4 исходные таблицы (`contracts`, `internet`, `personal`, `phone`) через `LEFT JOIN` по ключевому полю `customer_id`.
* **Защита от Data Leakage:** На этапе запроса и постобработки исключаются все технические индексы (колонка `index`) и суррогатные ключи, чтобы избежать переобучения модели на нефизических признаках.

### 2. Трансформация и генерация таргета (`transform`)
* **Обработка пропусков:** В признаке `end_date` (дата окончания контракта) строковые значения `'No'` приводятся к системным пропускам `None` / `NaN`. Это необходимо для корректной работы алгоритмов с временными рядами.
* **Генерация целевой переменной (`target`):** Создается бинарный признак оттока:
  * `1` — контракт завершен (клиент ушел).
  * `0` — в поле даты окончания стоит пропуск (клиент активен).

### 3. Загрузка данных (`load`)
* **Приемник:** Персональная база данных PostgreSQL, таблица по умолчанию: `users_churn`.
* **Режим записи:** Исполняется стратегия `replace` (полная перезапись таблицы при каждом запуске пайплайна) для предотвращения дублирования данных при повторных итерациях обучения. Системные индексы `pandas.DataFrame` в базу не сохраняются.

## 🚀 Требования и окружение

Скрипт должен запускаться строго внутри изолированного виртуального окружения Python:

### Зависимости
* `pandas` — для манипуляции данными в памяти.
* `sqlalchemy` — в качестве ORM-движка для безопасного подключения к СУБД.
* `psycopg2-binary` — системный драйвер для работы со спецификой PostgreSQL.
* `numpy` — для быстрой векторной предобработки данных.

### Команда для установки зависимостей:
```bash
pip install pandas sqlalchemy psycopg2-binary numpy
```

## 📦 Схема финального датасета

В результате работы ETL формируется плоская таблица со следующей структурой:
* `customer_id` — уникальный идентификатор клиента.
* `begin_date` / `end_date` — временные метки жизненного цикла контракта.
* **Блок признаков контракта:** тип оплаты, бумажный чек, общие и ежемесячные траты.
* **Блок признаков интернет-услуг:** тип подключения, безопасность, бэкапы, стриминг.
* **Блок персональных данных:** пол, пенсионный статус, наличие партнера/иждивенцев.
* **Блок телефонии:** многоканальные линии.
* `target` — **Целевая переменная** (0 или 1) для обучения ML-модели.
